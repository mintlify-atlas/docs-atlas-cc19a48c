This guide walks you through setting up NBAPredict, preparing the data, training models, and viewing results.

## Prerequisites

You'll need:
- Python 3.10 or higher
- A Kaggle account and API credentials
- Jupyter Notebook or VS Code with Jupyter extension

## Installation

<Steps>
  <Step title="Clone the repository">
    Clone the NBAPredict repository to your local machine:
    
    ```bash
    git clone https://github.com/Techdude01/NBAPredict.git
    cd NBAPredict
    ```
  </Step>

  <Step title="Create a virtual environment">
    Create and activate a Python virtual environment:
    
    ```bash
    python -m venv venv
    source venv/bin/activate  # On Windows: venv\Scripts\activate
    ```
  </Step>

  <Step title="Install dependencies">
    Install all required packages from requirements.txt:
    
    ```bash
    pip install -r requirements.txt
    ```
    
    This installs:
    - `pandas` and `numpy` for data manipulation
    - `kaggle` for downloading NBA datasets
    - `tensorflow` and `scikit-learn` for machine learning
    - `matplotlib` and `seaborn` for visualization
    - `python-dotenv` for environment management
  </Step>
</Steps>

## Configure Kaggle API

<Steps>
  <Step title="Get your Kaggle API credentials">
    1. Go to [Kaggle](https://www.kaggle.com/) and sign in
    2. Navigate to your account settings
    3. Scroll to the "API" section
    4. Click "Create New API Token"
    5. Save the downloaded `kaggle.json` file
    
    <Note>
    The `kaggle.json` file contains your username and API key. Keep this file secure and never commit it to version control.
    </Note>
  </Step>

  <Step title="Create environment file">
    Copy the example environment file:
    
    ```bash
    cp .env.example .env
    ```
    
    Edit `.env` and add your Kaggle credentials:
    
    ```bash
    KAGGLE_USERNAME=your_username
    KAGGLE_KEY=your_api_key
    ```
    
    <Warning>
    Make sure `.env` is in your `.gitignore` file to prevent accidentally committing your credentials.
    </Warning>
  </Step>
</Steps>

## Prepare the data

<Steps>
  <Step title="Open the data preparation notebook">
    Launch Jupyter and open `NBAPredict.ipynb`:
    
    ```bash
    jupyter notebook NBAPredict.ipynb
    ```
    
    Or open it in VS Code with the Jupyter extension.
  </Step>

  <Step title="Load Kaggle credentials">
    The first cell loads your credentials from the `.env` file:
    
    ```python
    import os
    from dotenv import load_dotenv

    # Load credentials from .env file
    load_dotenv()

    # Set Kaggle credentials as environment variables
    os.environ['KAGGLE_USERNAME'] = os.getenv('KAGGLE_USERNAME')
    os.environ['KAGGLE_KEY'] = os.getenv('KAGGLE_KEY')

    print("Kaggle credentials loaded!")
    ```
  </Step>

  <Step title="Download the NBA dataset">
    Run the cell that downloads historical NBA data from Kaggle:
    
    ```python
    from kaggle.api.kaggle_api_extended import KaggleApi

    # Initialize and authenticate
    api = KaggleApi()
    api.authenticate()

    # Download NBA dataset
    dataset_name = "eoinamoore/historical-nba-data-and-player-box-scores"
    download_path = "./data"
    os.makedirs(download_path, exist_ok=True)

    api.dataset_download_files(dataset_name, path=download_path, unzip=True)
    print(f"Dataset downloaded to {download_path}")
    ```
    
    <Tip>
    The dataset includes game logs from 1946 to present, with team statistics, player box scores, and historical records.
    </Tip>
  </Step>

  <Step title="Process the data">
    Continue running cells to clean and aggregate the data. The notebook will:
    
    - Parse game dates and extract season information
    - Map team relocations to current franchises
    - Aggregate per-team-season statistics
    - Create train/validation/test splits
    - Generate multiple feature transformations (baseline, polynomial, PCA)
    
    The processed data is saved to `./data/processed/`:
    
    ```
    data/processed/
    ├── X_train_scaled.npy
    ├── X_train_poly2.npy
    ├── X_train_poly3.npy
    ├── X_train_pca.npy
    ├── X_val_scaled.npy
    ├── X_test_scaled.npy
    └── y_train.npy, y_val.npy, y_test.npy
    ```
  </Step>
</Steps>

## Train models

You can train three different model types: logistic regression, K-nearest neighbors, and neural networks. Each model is trained on multiple feature transformations.

<Steps>
  <Step title="Train logistic regression">
    Open and run `LogPredict.ipynb`:
    
    ```python
    # Load processed data
    X_train = np.load('data/processed/X_train_scaled.npy')
    y_train = np.load('data/processed/y_train.npy')

    # Augment X with bias term
    def augment_X(X):
        ones = np.ones((X.shape[0], 1))
        return np.hstack([ones, X])

    X_train_aug = augment_X(X_train)

    # Train with gradient ascent
    w, log_likelihood_values = Gradient_Ascent(
        X_train_aug, 
        y_train.reshape(-1, 1),
        learning_rate=0.01,
        num_iters=10000,
        lambda_reg=0.1
    )
    ```
    
    <Note>
    The logistic regression model is implemented from scratch using gradient ascent with L2 regularization. This mirrors typical machine learning coursework.
    </Note>
  </Step>

  <Step title="Train KNN models">
    Open and run `KNNPredict.ipynb` to train K-nearest neighbors models with grid search:
    
    ```python
    from sklearn.neighbors import KNeighborsClassifier
    from sklearn.model_selection import GridSearchCV

    # Define parameter grid
    param_grid = {
        'n_neighbors': [3, 5, 7, 9, 11],
        'weights': ['uniform', 'distance'],
        'metric': ['euclidean', 'manhattan']
    }

    # Grid search
    knn = KNeighborsClassifier()
    grid_search = GridSearchCV(knn, param_grid, cv=5, scoring='roc_auc')
    grid_search.fit(X_train, y_train)
    ```
  </Step>

  <Step title="Train neural networks">
    Open and run `NeuralNetworkPredict.ipynb` to train TensorFlow/Keras models:
    
    ```python
    import tensorflow as tf
    from tensorflow import keras

    # Build a simple feedforward network
    model = keras.Sequential([
        keras.layers.Dense(64, activation='relu', input_shape=(X_train.shape[1],)),
        keras.layers.Dropout(0.3),
        keras.layers.Dense(32, activation='relu'),
        keras.layers.Dropout(0.3),
        keras.layers.Dense(1, activation='sigmoid')
    ])

    model.compile(
        optimizer='adam',
        loss='binary_crossentropy',
        metrics=['accuracy', 'AUC']
    )

    # Train the model
    history = model.fit(
        X_train, y_train,
        validation_data=(X_val, y_val),
        epochs=100,
        batch_size=32,
        verbose=1
    )
    ```
    
    <Tip>
    Neural networks typically outperform the baseline models on AUC, but all three approaches are competitive and sensitive to feature engineering choices.
    </Tip>
  </Step>
</Steps>

## View results

<Steps>
  <Step title="Run model comparison">
    Open and run `ModelComparison.ipynb` to compare all trained models:
    
    The notebook loads the best run from each model family and reports:
    - Accuracy
    - Precision
    - Recall
    - F1 score
    - AUC (Area Under ROC Curve)
    
    You'll see visualizations comparing model performance across different feature transformations.
  </Step>

  <Step title="Interpret the results">
    The comparison notebook highlights:
    
    - **Best overall model**: Neural networks slightly outperform on AUC
    - **Feature importance**: Which transformations improve predictions
    - **Trade-offs**: Precision vs. recall for different model types
    
    All three model families are competitive, with neural networks showing a slight edge on the AUC metric.
  </Step>
</Steps>
