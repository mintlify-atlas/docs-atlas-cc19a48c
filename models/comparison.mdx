# Model Comparison

## Executive Summary

This page compares three machine learning approaches for predicting NBA playoff team qualification:

1. **Logistic Regression** - Linear baseline with L2 regularization
2. **K-Nearest Neighbors** - Distance-based non-parametric classifier
3. **Neural Network** - Deep learning with multiple hidden layers

<Note>
**Winner:** The **Neural Network** achieved the highest AUC (0.9824) with baseline features and L2-0.01 regularization, making it the recommended model for production deployment.
</Note>

## Performance Comparison

### Test Set Metrics (Best Models)

| Model | Feature Transform | Accuracy | Precision | Recall | F1 Score | AUC |
|-------|-------------------|----------|-----------|--------|----------|-----|
| **Neural Network** | Baseline | 0.8750 | 0.8936 | 0.9844 | 0.8936 | **0.9824** |
| Logistic Regression | Baseline | **0.8917** | 0.8493 | 0.9688 | **0.9051** | 0.9754 |
| K-Nearest Neighbors | Polynomial 3 | 0.8167 | 0.7500 | **0.9844** | 0.8514 | 0.9660 |

<AccordionGroup>
  <Accordion title="Metric Explanations">
    - **Accuracy:** Percentage of correct predictions (playoff or non-playoff)
    - **Precision:** Of teams predicted as playoff, what percentage actually made it?
    - **Recall:** Of actual playoff teams, what percentage did we identify?
    - **F1 Score:** Harmonic mean of precision and recall (balanced metric)
    - **AUC:** Area Under ROC Curve (discrimination ability, 0.5=random, 1.0=perfect)
  </Accordion>
</AccordionGroup>

## Computational Complexity

### Training Time

<Tabs>
  <Tab title="Logistic Regression">
    **Training:** ~5 seconds (10,000 iterations)
    
    **Prediction:** Instant (matrix multiplication)
    
    **Complexity:** O(n × d × iterations)
    - n = number of samples
    - d = number of features
  </Tab>
  
  <Tab title="K-Nearest Neighbors">
    **Training:** Instant (lazy learner)
    
    **Prediction:** ~2 seconds (distance calculations)
    
    **Complexity:** O(n × d) per prediction
    - Scales linearly with training set size
    - Computationally expensive for large datasets
  </Tab>
  
  <Tab title="Neural Network">
    **Training:** ~30 seconds (with early stopping)
    
    **Prediction:** Instant (forward pass)
    
    **Complexity:** O(epochs × batch_size × layers × units)
    - Training requires GPU for large models
    - Prediction is very fast
  </Tab>
</Tabs>

## Model Interpretability

<AccordionGroup>
  <Accordion title="Logistic Regression - Highest Interpretability">
    **Interpretation Method:** Weight coefficients
    
    **Example:**
    ```python
    Positive Weights:
    - Win%: +2.34 (stronger playoff predictor)
    - OffRtg: +1.87 (offensive efficiency important)
    
    Negative Weights:
    - Losses: -1.92 (more losses = less likely playoff)
    ```
    
    **Advantages:**
    - Direct feature importance
    - Understand prediction logic
    - Regulatory compliance
  </Accordion>
  
  <Accordion title="K-Nearest Neighbors - Medium Interpretability">
    **Interpretation Method:** Nearest neighbor examples
    
    **Example:**
    ```python
    Prediction: Playoff Team
    
    Nearest Neighbors:
    1. 2019 Bucks (Playoff) - Distance: 0.23
    2. 2018 Warriors (Playoff) - Distance: 0.31
    3. 2020 Lakers (Playoff) - Distance: 0.38
    ```
    
    **Advantages:**
    - Example-based explanations
    - Intuitive for humans
    - Can inspect similar historical teams
  </Accordion>
  
  <Accordion title="Neural Network - Lowest Interpretability">
    **Interpretation Method:** Black box (requires additional tools)
    
    **Techniques:**
    - SHAP (Shapley Additive Explanations)
    - LIME (Local Interpretable Model-agnostic Explanations)
    - Gradient-based attribution
    
    **Challenges:**
    - Non-linear transformations
    - Multiple layer interactions
    - Requires specialized interpretation libraries
  </Accordion>
</AccordionGroup>

## Strengths and Weaknesses

### Logistic Regression

<CardGroup cols={2}>
  <Card title="Strengths" icon="circle-check">
    - Highest interpretability
    - Fast training and prediction
    - Probabilistic outputs
    - Requires minimal tuning
    - Excellent F1 score (0.9051)
  </Card>
  <Card title="Weaknesses" icon="circle-xmark">
    - Linear decision boundary
    - Cannot capture complex interactions
    - Assumes feature independence
    - Lower AUC than neural network
  </Card>
</CardGroup>

### K-Nearest Neighbors

<CardGroup cols={2}>
  <Card title="Strengths" icon="circle-check">
    - Highest recall (98.44%)
    - No training phase
    - Non-parametric flexibility
    - Example-based predictions
    - No assumptions about data distribution
  </Card>
  <Card title="Weaknesses" icon="circle-xmark">
    - Slow prediction time
    - Memory intensive
    - Sensitive to feature scaling
    - Lowest precision (0.75)
    - Curse of dimensionality
  </Card>
</CardGroup>

### Neural Network

<CardGroup cols={2}>
  <Card title="Strengths" icon="circle-check">
    - **Highest AUC (0.9824)**
    - Learns non-linear patterns
    - Automatic feature interactions
    - Excellent recall (98.44%)
    - Best discrimination ability
  </Card>
  <Card title="Weaknesses" icon="circle-xmark">
    - Black box (least interpretable)
    - Requires hyperparameter tuning
    - Longer training time
    - Risk of overfitting
    - Needs more data for deep architectures
  </Card>
</CardGroup>

## Use Case Recommendations

<Steps>
  <Step title="Use Neural Network When:">
    - **Maximizing AUC/discrimination** is the priority
    - You have **sufficient computational resources**
    - **Interpretability is not critical**
    - You need **high recall** with good precision
    - **Production deployment** with offline training
  </Step>
  
  <Step title="Use Logistic Regression When:">
    - **Interpretability** is important
    - You need **fast training/prediction**
    - **Regulatory compliance** requires explainability
    - You want a **strong baseline** model
    - **Resource constraints** (embedded systems)
  </Step>
  
  <Step title="Use K-Nearest Neighbors When:">
    - **Avoiding false negatives** is critical
    - You want **example-based predictions**
    - **No training phase** is required
    - You have **polynomial features** available
    - **Online learning** (update by adding examples)
  </Step>
</Steps>

## Final Recommendation

<Note>
**Production Deployment:** Use the **Neural Network (2-Layer [128, 64])** with **L2 regularization (λ=0.01)** and **baseline scaled features**.
