The `KNNPredict.ipynb` notebook trains K-nearest neighbors (KNN) classifiers using scikit-learn. It performs exhaustive grid search over K values, distance metrics, weighting schemes, and feature transformations to find the optimal configuration.

## Purpose

This notebook explores KNN as a non-parametric approach to playoff prediction:

- Tests multiple K values (number of neighbors)
- Compares Euclidean vs Manhattan distance metrics
- Evaluates uniform vs distance-weighted voting
- Identifies which feature transformation works best for KNN

## Implementation

### Training a KNN model

```python
from sklearn.neighbors import KNeighborsClassifier

# Create KNN classifier
knn = KNeighborsClassifier(
    n_neighbors=21,
    metric='manhattan',
    weights='distance'
)

# Train on scaled features
knn.fit(X_train_scaled, y_train)

# Predict on test set
y_pred = knn.predict(X_test_scaled)
y_proba = knn.predict_proba(X_test_scaled)[:, 1]
```

### Distance metrics

<Tabs>
  <Tab title="Euclidean">
    **Euclidean distance** (L2 norm):
    
    d(x, y) = √(Σ(x_i - y_i)²)
    
    Measures straight-line distance in feature space. Sensitive to feature scale—requires standardization.
    
    ```python
    knn = KNeighborsClassifier(n_neighbors=5, metric='euclidean')
    ```
  </Tab>
  
  <Tab title="Manhattan">
    **Manhattan distance** (L1 norm):
    
    d(x, y) = Σ|x_i - y_i|
    
    Measures grid-like distance (sum of absolute differences). More robust to outliers than Euclidean distance.
    
    ```python
    knn = KNeighborsClassifier(n_neighbors=5, metric='manhattan')
    ```
  </Tab>
</Tabs>

### Weighting schemes

<AccordionGroup>
  <Accordion title="Uniform weighting">
    All K neighbors vote equally regardless of distance:
    
    ```python
    knn = KNeighborsClassifier(n_neighbors=5, weights='uniform')
    ```
    
    Works well when neighbors are roughly equidistant. Simpler but less flexible.
  </Accordion>
  
  <Accordion title="Distance weighting">
    Closer neighbors have more influence (weighted by inverse distance):
    
    ```python
    knn = KNeighborsClassifier(n_neighbors=5, weights='distance')
    ```
    
    Weight = 1 / distance
    
    Reduces impact of distant neighbors. Generally performs better on complex decision boundaries.
  </Accordion>
</AccordionGroup>

## Best model results

The optimal configuration:

- **K**: 21 neighbors
- **Distance metric**: Manhattan
- **Weighting**: Distance-based
- **Feature transformation**: Polynomial degree 3

### Test set performance

| Metric | Value |
|--------|-------|
| Accuracy | 81.67% |
| Precision | 75.00% |
| Recall | **98.44%** |
| F1 Score | 85.14% |
| AUC | 96.60% |

<Note>
  KNN achieves the highest recall (98.44%) among all three models, meaning it rarely misses actual playoff teams. However, it has the lowest precision (75%)—it flags more false positives.
</Note>

## Performance by K value

Validation accuracy vs K (using Manhattan distance, distance weighting, Poly3 features):

| K | Validation Accuracy |
|---|---------------------|
| 1 | 85.83% |
| 3 | 87.50% |
| 5 | 88.33% |
| 7 | 89.17% |
| 9 | 89.17% |
| 11 | 90.00% |
| 15 | 90.83% |
| **21** | **91.67%** |

<Tip>
  Higher K values (15–21) perform best on this dataset. Lower K values (1–5) overfit to training data and are sensitive to noise.
</Tip>

## Feature transformation impact

KNN performance by feature type (K=21, Manhattan, distance-weighted):

| Transformation | Test AUC |
|----------------|----------|
| Baseline scaled | 94.12% |
| Polynomial degree 2 | 95.48% |
| **Polynomial degree 3** | **96.60%** |
| PCA | 92.85% |

**Key insight**: Unlike logistic regression and neural networks, KNN benefits significantly from polynomial features. The explicit feature interactions help KNN identify similar teams in high-dimensional space.

## Strengths and weaknesses

<CardGroup cols={2}>
  <Card title="Strengths" icon="circle-check">
    - No training phase (lazy learning)
    - Non-parametric (no assumptions about data distribution)
    - Excellent recall (98.44%)
    - Works well with polynomial features
  </Card>
  
  <Card title="Weaknesses" icon="circle-xmark">
    - Slow prediction (must compute distances to all training samples)
    - Memory-intensive (stores entire training set)
    - Lower precision (75%)
    - Sensitive to feature scaling
  </Card>
</CardGroup>
